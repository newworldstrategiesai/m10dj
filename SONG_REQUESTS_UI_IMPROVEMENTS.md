# Song Requests Admin UI Improvements

## Proposed Enhancements

### 1. **Enhanced Filtering System**
   - **Event Filter**: Dropdown to filter by event QR code
   - **Date Range Filter**: Select start and end dates
   - **Request Type Filter**: Filter by song requests vs shoutouts
   - **Payment Status Filter**: More granular payment filtering
   - **Combined Filters**: Apply multiple filters simultaneously

### 2. **Request Detail Modal**
   - Click on any request to see full details in a modal
   - Quick actions within the modal
   - No need to leave the list view

### 3. **View Toggle**
   - **Cards View**: Better for mobile, shows more visual info
   - **List/Table View**: Current table view (better for desktop)
   - Toggle button to switch between views

### 4. **Event Grouping**
   - Option to group requests by event QR code
   - Collapsible event sections
   - Event-level statistics

### 5. **Bulk Actions**
   - Select multiple requests (checkbox column)
   - Bulk update status
   - Bulk mark as paid
   - Bulk export

### 6. **Better Mobile Responsiveness**
   - Cards view optimized for mobile
   - Touch-friendly buttons
   - Responsive table with horizontal scroll on mobile

### 7. **Sorting Options**
   - Sort by date (newest/oldest)
   - Sort by amount (highest/lowest)
   - Sort by priority (fast-track first)
   - Sort by status

### 8. **Export Functionality**
   - Export to CSV
   - Export to playlist format (for DJ software)
   - Export filtered results only

### 9. **Quick Actions**
   - Keyboard shortcuts for common actions
   - Right-click context menu
   - Quick status update buttons

### 10. **Real-time Updates**
   - Polling for new requests (optional)
   - Visual indicators for new requests
   - Auto-refresh toggle

## Priority Implementation Order

1. ✅ Enhanced filtering (event, date range, type)
2. ✅ Request detail modal
3. ✅ View toggle (cards/list)
4. ✅ Better mobile responsiveness
5. ⏳ Event grouping
6. ⏳ Bulk actions
7. ⏳ Export functionality
8. ⏳ Sorting options
